Tecnologias Utilizadas

  • HTML5 (3 páginas (index, stats, about), semântica acessível, formulários, datalist, modais e cards)
  • CSS3 (layout responsivo com Grid e Flexbox, variáveis CSS, backdrop-filter, navbar mobile, sem frameworks externas)
  • JavaScript (chamadas AJAX assíncronas via fetch + async/await, DOM, validação de formulários, feedback ao utilizador (toasts de sucesso e erro), operações CRUD)
  • API (Node/Express) com operações GET/POST/PUT/DELETE

Dificuldades Sentidas

  1. Garantir responsividade sem bibliotecas externas.
  2. Validação de dados em dois lugares: compreender as regras entre frontend e backend, garantindo resposta ao utilizador, segurança dos dados e consistência.
  3. Tratar erros e estados de loading de forma consistente.
  4. Manter o código organizado (helpers reutilizáveis e sem repetição).
  5. Assincronia em JavaScript: entender o funcionamento do código quando há async/await e saber identificar potenciais problemas.
  6. Alinhar a API com as necessidades do dashboard (prioridades e recentes).
  7. Comportamento do DOM: compreender como alterações ao HTML podem interferir com outros elementos da página, e como o browser processa eventos e estilos em simultâneo.
